Configuration & Options

Below are the available attributes to configure the directive.

input-model
required

This is the data you pass to the directive. You're allowed to pass 4 different types of data. An array of nested objects, a specially crafted array of objects, suitable to be used by the directive's engine, bypassing the check-prerequisites and the to-internal calls from the data converter module, thus allowing you to squeeze some extra performance, a literal JSON string or a URL.

  • Array of nested objects

    						
input-model="input_data"
						
					
    						
$scope.input_data = [
	{ text: 'A', value: 'a', id: 1, children: 'foo'},
	{ text: 'B', value: 'b', id: 2, children: [
		{ text: 'C', value: 'c', id: 3, checked: true}
	]},
	{ text: 'D', value: 'd', id: 4, children: [
		{ text: 'E', value: 'e', id: 5, children: []},
		{ text: 'F', value: 'f', id: 6, checked: true}
	]}
]

  • Specially crafted array of objects

    						
input-model="input_data"
						
					
    						
$scope.input_data = [
	{
		text: 'A', value: 'a', id: 1, checked: false, open: false,
		$ams_order: 1, $ams_parents_id: [], $ams_children_nodes: 0,
		$ams_children_leafs: 0,$ams_level: 0, $ams_tree_visibility: true,  $ams_checked_children: 0
	},
	{
		text: 'B', value: 'b', id: 2, checked: 1, open: true,
		$ams_order: 2, $ams_parents_id: [], $ams_children_nodes: 0,
		$ams_children_leafs: 1,$ams_level: 0, $ams_tree_visibility: true,  $ams_checked_children: 1
	},
	{
		text: 'C', value: 'c', id: 3, checked: true, open: false,
		$ams_order: 3, $ams_parents_id: [2], $ams_children_nodes: 0,
		$ams_children_leafs: 0,$ams_level: 1, $ams_tree_visibility: true,  $ams_checked_children: 0
	},
	{
		text: 'D', value: 'd', id: 4, checked: 0, open: false,
		$ams_order: 4, $ams_parents_id: [], $ams_children_nodes: 0,
		$ams_children_leafs: 2,$ams_level: 0, $ams_tree_visibility: true,  $ams_checked_children: 1
	},
	{
		text: 'E', value: 'e', id: 5, checked: false, open: false,
		$ams_order: 5, $ams_parents_id: [4], $ams_children_nodes: 0,
		$ams_children_leafs: 0,$ams_level: 1, $ams_tree_visibility: false, $ams_checked_children: 0
	},
	{
		text: 'F', value: 'f', id: 6, checked: true, open: false,
		$ams_order: 6, $ams_parents_id: [4], $ams_children_nodes: 0,
		$ams_children_leafs: 0,$ams_level: 1, $ams_tree_visibility: false, $ams_checked_children: 0
	 }
]

  • Literal JSON string. Note that the JSON literal is delimited by single quotes

    						
input-model='[{"text":"A","value":"a","id":1},
{"text":"B","value":"b","id":2,"open":true,"children":[{"text":"C","value":"c","id":3,"checked":true}]},
{"text":"D","value":"d","id":4,"children":[{"text":"E","value":"e","id":5},{"text":"F","value":"f","id":6,"checked":true}]}]'

  • Literal URL that will be fetched with a GET request. Note that the URL is delimited by single quotes inside double quotes

    						
input-model="'http://foo.com/bar.json'"
output-model
required

The model where the output of AMS will be stored. This can't be anything else but a variable. 

				
output-model="output_data"
				
			
				
$scope.output_data = []
output-type

This alows you to choose the desired output format to be used. The possible values are:

  • objects default - The output will be an array of objects 
    							
output-type="objects"
							
						
    							
input = [
	{a: 'foo1', b: 'bar1', c: 42},
	{a: 'foo2', b: 'bar2', c: 43},
	{a: 'foo3', b: 'bar3', c: 44}
];

output = [
	{a: 'foo1', b: 'bar1', c: 42},
	{a: 'foo2', b: 'bar2', c: 43},
	{a: 'foo3', b: 'bar3', c: 44}
];
  • arrays - The output will be an array of arrays. Note that an object doesn't have an "order", so values could be returned in the order you're not expecting. Check output-keys for a solution. 
    							
output-type="arrays"
							
						
    							
input = [
	{a: 'foo1', b: 'bar1', c: 42},
	{a: 'foo2', b: 'bar2', c: 43},
	{a: 'foo3', b: 'bar3', c: 44}
];

output = [
	['foo1', 'bar1', 42],
	['foo2', 'bar2', 43],
	['foo3', 'bar3', 44]
];
  • object - The output will be a single object. This makes sense if you're limiting the maximum selected items to 1. If more than 1 item is selected, the object will contain they keys/values of the first selected item. 
    							
output-type="object"
							
						
    							
input = [
	{a: 'foo1', b: 'bar1', c: 42},
	{a: 'foo2', b: 'bar2', c: 43},
	{a: 'foo3', b: 'bar3', c: 44}
];

output = {a: 'foo1', b: 'bar1', c: 42};
  • array - The output will be a single array. This makes sense if you're limiting the maximum selected items to 1. If more than 1 item is selected, the array will contain the values of the first selected item. 
    							
output-type="array"
							
						
    							
input = [
	{a: 'foo1', b: 'bar1', c: 42},
	{a: 'foo2', b: 'bar2', c: 43},
	{a: 'foo3', b: 'bar3', c: 44}
];

output = ['foo1', 'bar1', 42];
  • value - The output will be a single value, the value of the key that is passed (check output-keys), in the first selected item. If no key is passed, the first available value in the first selected item will be returned. 
    							
output-type="value"
							
						
    							
input = [
	{a: 'foo1', b: 'bar1', c: 42},
	{a: 'foo2', b: 'bar2', c: 43},
	{a: 'foo3', b: 'bar3', c: 44}
];

output = 'foo1';

output-keys

This is used to specify the desired keys/values that should be available in the output model. If no keys are specified, everything will be returned. Note that the order in which the keys are passed is used/conserved when creating the output data, which is very useful when output-type is array/arrays. 

				
output-keys="c, a"
output-type="array"
				
			
				
input = [
	{a: 'foo1', b: 'bar1', c: 42},
	{a: 'foo2', b: 'bar2', c: 43},
	{a: 'foo3', b: 'bar3', c: 44}
];

output = [42, 'foo1'];
output-filter

You might want the output model to contain only certain items, for example, only the checked leafs or only the checked nodes. You must know that a:

  • leaf - is an item without children
  • node - is an item with children

and that:

  • leafs - can be checked or unchecked
  • nodes - can be checked, unchecked or mixed, that last one meaning that the node has 1 or more checked children, but not all of them are checked.

Here is a complete list of the output filters you can apply:

  • leafs default - Return only the checked leafs 
    							
output-filter="leafs"
  • leafs_mixed_nodes - Return only the checked leafs and mixed nodes 
    							
output-filter="leafs_mixed_nodes"
  • leafs_checked_nodes - Return only the checked nodes 
    							
output-filter="leafs_checked_nodes"
  • leafs_mixed_checked_nodes - Return the checked leafs and the mixed and checked nodes 
    							
output-filter="leafs_mixed_checked_nodes"
  • midex_nodes - Return only the mixed nodes 
    							
output-filter="midex_nodes"
  • checked_nodes - Return only the checked nodes 
    							
output-filter="checked_nodes"
  • mixed_checked_nodes - Return only the mixed and checked nodes 
    							
output-filter="mixed_checked_nodes"

id-property default is 'id'

The property that will be used to acces the object's ID. 

				
id-property="my-custom-id-property"
open-property default is 'open'

The property that will be used to check the if the object is open or closed. 

				
open-property="my-custom-open-property"
checked-property default is 'checked'

The property that will be used to check the if the object is checked or unchecked. 

				
checked-property="my-custom-checked-property"
children-property default is 'children'

The property that will be used to access the object's children. 

				
children-property="my-custom-children-property"
do-not-check-data default is 'false'

If true, will skip the check_prerequisites step. Make sure you understand what this means, read carefully Under the hood. 

				
do-not-check-data="true"
do-not-convert-data default is 'false'

If true, will skip the to_internal step. Make sure you understand what this means, read carefully Under the hood. 

				
do-not-convert-data="true"
max-checked-leafs default is -1 (unlimited)

This will allow you to limit the number of checked leafs. 

				
max-checked-leafs="5"
dropdown-label

You can set the text that appears in the dropdown button itself. You're allowed to use HTML and CSS. You can use AMS's constant values that will be interpolated with the <# and #> symbols. You can also use Angular's inline expressions with the <[ and ]> interpolate symbols. 

				
dropdown-label="<[
	<# angularMultiSelectConstants.INTERNAL_STATS_TOTAL_LEAFS #> === undefined ? 'No available items' :
		<# angularMultiSelectConstants.INTERNAL_STATS_CHECKED_LEAFS #> === 0 ? 'No checked items' :
			<# angularMultiSelectConstants.INTERNAL_STATS_CHECKED_LEAFS #> + ' checked ' + (<# angularMultiSelectConstants.INTERNAL_STATS_CHECKED_LEAFS #> === 1 ? 'item': 'items')
]>"

Outputs:

  • Empty output model - 'No available items'
  • No checked items - 'No checked items'
  • 1 checked item - '1 checked item'
  • Multiple checked items - 'N checked item'

Starting with 6.1.0 you can also pass a template string to a special filter, which will be interpolated with each of the items in your output model. 

				
dropdown-label="<[ '<(text)>' | outputModelIterator : this ]>"
dropdown-label="<[ '<(text)> - <(value)>' | outputModelIterator : this : ' | ' ]>"
dropdown-label="<[ '<(this[0])> - <(this[1])>' | outputModelIterator : this ]>" // output-type="arrays"
dropdown-label="<[ '' | outputModelIterator : this ]>" // output-type="value"

Output: 

				
C,F,S,Y,Z
C - c | F - f | S - s | Y - y | Z - y
C - c,F - f,S - s,Y - y,Z - z
C
node-label

You can set the text that appears in each node. You're allowed to use HTML and CSS. You can use AMS's constant values that will be interpolated with the <# and #> symbols. You can also use Angular's inline expressions with the <[ and ]> interpolate symbols. Those symbols are also used to access each item's properties. 

				
node-label="<[ text ]> (checked
	<# angularMultiSelectConstants.INTERNAL_KEY_CHECKED_CHILDREN #> /
	<# angularMultiSelectConstants.INTERNAL_KEY_CHILDREN_LEAFS #>)"

Outputs:

  • Foo bar (checked 0 / 2)
  • Bar (checked 3 / 7)
  • Foo bar (checked 8 / 8)

leaf-label

You can set the text that appears in each leaf. You're allowed to use HTML and CSS. You can use AMS's constant values that will be interpolated with the <# and #> symbols. You can also use Angular's inline expressions with the <[ and ]> interpolate symbols. Those symbols are also used to access each item's properties. 

				
leaf-label="<[ text ]> (<[ value ]>)"

Outputs:

  • Foo (1)
  • Bar (2)
  • Foo bar (3)

preselect

If you can't mark an item in the input model as checked, you can use this to check it. This attribute accepts a pair of string separeted by a comma. The first one is the key, while the second one is the value. The first element that matches the passed pair of values will be marked as checked. 

				
preselect="foo, 42"
hide-helpers

The search input is automatically hidden if the search-field attribute is empty, but the rest of the helpers (Check all (check_all), Uncheck all (check_none), Reset (reset)) are always visible. You can hide those with this attribute, by passing the names of the helpers you want to hide. 

				
hide-helpers="check_all, reset"
search-field

Allows to specify the name of the property that should be used to perform the search operations. 

				
search-field="foo"
class

Allows to set a custom class to the directive itself 

				
class="my-class"
name

This will force a specific internal name used for LokiJS and for the debug messages. Normally you won't need to use this. 

				
name="my-name"
debug

Enables debugging mode 

				
debug="true"
