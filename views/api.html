API

Controlling AMS programmatically

AMS allows to be controlled programmatically with a signals-based API that accepts the following event names:

  • ams_do_check_all 
    						
$rootScope.$broadcast('ams_do_check_all', {
	name: 'my ams'
});
  • ams_do_uncheck_all 
    						
$rootScope.$broadcast('ams_do_uncheck_all', {
	name: 'my ams'
});
  • ams_do_reset 
    						
$rootScope.$broadcast('ams_do_reset', {
	name: 'my ams'
});
  • ams_do_toggle_open_node 
    						
$rootScope.$broadcast('ams_do_toggle_open_node', {
	name: 'my ams',
	item: {id: 4}
});
  • ams_do_toggle_check_node 
    						
$rootScope.$broadcast('ams_do_toggle_check_node', {
	name: 'my ams',
	item: {id: 4}
});

As you can see, the important argument is the args one. That argument is an object containing 2 keys.
  • name - the name you (or AMS) assigned to the directive or '*'. That allows you to controll only a specific AMS instance or all instances.
  • item - the item that you want to apply the action on. Normally you'd want to pass your id property and the value, but you can pass anything you want. All the keys/values will be passed to Loki and the first matching result will be used. 
    					
$rootScope.$broadcast('ams_do_toggle_check_node', {
	name: 'my ams',
	item: {name: 'John', age: 42}
});
