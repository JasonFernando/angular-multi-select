Events

Listening for events

AMS will broadcast events whenever an item is un/checked or opened/closed. The names of the events are:

  • ams_toggle_check_node
  • ams_toggle_open_node

			
$scope.$on('ams_toggle_check_node', function(event, args) {
	console.log(event, args);
});

$scope.$on('ams_toggle_open_node', function(event, args) {
	console.log(event, args);
});
That will output something like 
			
Object {name: "ams_toggle_open_node", targetScope: m, defaultPrevented: false, currentScope: b},
Object {name: "angular-multi-select-14609208330.14700627423064772", item: {"text":"A","value":"a","id":1,"open":false,"checked":1}}

Object {name: "ams_toggle_open_node", targetScope: m, defaultPrevented: false, currentScope: b},
Object {name: "angular-multi-select-14609208330.14700627423064772", item: {"text":"A","value":"a","id":1,"open":false,"checked":1}}
As you can see, the important argument is the args one. That argument is an object containing 2 keys.
  • name - the name you (or AMS) assigned to the directive
  • item - the item that caused the broadcast before the actual change
