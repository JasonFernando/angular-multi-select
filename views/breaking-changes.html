Breaking Changes

Breaking changes since v6.x

  • This is a complete rewrite of the project, please read the documentation carefully.
  • All attributes have been changed.
  • Document hacks are not required anymore.
  • Syntax, classes, general HTML has been changed.

Breaking changes since v5.3.x

  • Item label and button label syntax changed.

Breaking changes since v5.2.x

  • CSS class names were changed/improved. If your code relied on class matching, please review it.

Breaking changes since v5.1.x

  • item-label now uses normal curly braces {{ foo }} instead of the previous {| foo |} from 5.0.x.

Breaking changes since v5.x.x

  • input-model can now be an array with nested arrays inside.
  • ouput-model is not required anymore if you don't need it.
  • group-property syntax has changed.
  • item-label syntax has changed.
  • button-label is not used anymore.

    • Please note that v5.x.x is the result of a fork of the project and the entire code was rewritten. It is possible that your code doesn't work with v5.x.x without some changes, so please test everything before putting this in production. Also, feel free to report any bugs.

Breaking changes since v3.x.x

  • input-model is now static (not dynamically updated), hence why we need output-model. On the plus side, you now can re-use the input model where necessary. Sorry, it's now back like version 2.x.x. As for now it's the only way to allow dynamic update into the input-model. If you need to re-use the input model, copy them into a new variable.

Breaking changes since v2.x.x

  • File name and the directive name have been changed. I am really sorry for this, but this is the only workaround to prevent wrong language statistic in Github (they don't count files whose name starts with "angular"). The repository name stays the same.
  • output-model is now required.
  • input-model is now static (not dynamically updated), hence why we need output-model. On the plus side, you now can re-use the input model where necessary.
  • default-label is deprecated. Custom text and translations can be done using the translation attribute.
